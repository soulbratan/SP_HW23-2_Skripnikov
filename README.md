## Примеры использования:
1. В пакете *'config'* расположены основные настройки проекта Django.
2. В пакете *'catalog'* расположены основные элементы приложения 'catalog'.
3. В пакете *'catalog'* создан модуль *'urls'* для настройки маршрутизации проекта.
4. В пакете *'catalog'* и модуле *'views'* определён контроллер 'home' для отображения домашней страницы.
5. В пакете *'catalog'* и модуле *'views'* определён контроллер 'home' для страницы с контактной информацией, а также обработки отправленных данных.
6. В пакете *'catalog'* директории *'templates'* подготовлены шаблоны домашней страницы 'home.html' и страницы контактов 'contacts.html'. Также в директории 'static' хранятся файлы для стилизации страниц.
7. В пакете *'catalog'* и модуле *'models'* определёны модели данных для Продуктов и Категорий.
8. Создана фикстура данных catalog.jsonю
9. Создан и настроен суперпользователь для работы с "Админкой"
10. Созданы базовые шаблоны для карточек с товарами, настроены фильтры, настроены медиа файлы.
11. Контроллеры приложения *'catalog'* переведены из FBV в CBV.
12. Создано приложение *'blog'* отображения и редактирования статей блога. Настроена маршрутизация (регистрация на адресс blogs/).
13. В приложении *'blog'* созданы базовый шаблон и шаблоны CRUD для блоговой записи.
14. В модуле *'forms'* созданы формы Django для модели Product. Для добавления и редактирования карточек товара реализован механизм CRUD.
15. Добавлена валидация по запрещённым словам и отрицательной цене.
16. Создано приложение *'users'* для работы с профилями пользователя. Также добавлено ограничение редактирования карточек товара для незарегистрированных пользователей.
17. При регистрации пользователя происходит верификация почты, после прохождения которой, профиль считается созданным.
18. Проработан функционал для проверки и разграничения прав доступа. Создана группа "Moderator of products", которая может менять значение "is_published" для публикации статьи.
19. Добавлены кастомные права к модели продукта (can_unpublish_product — может отменять публикацию продукта).
20. Определены права на удаление продукта только для владельца или модератора.
21. Установлена библиотека *redis* для кэширования данных. Настроено кеширование для страницы, которая отображает информацию об одном продукте.
22. Созданы сервисные функции в модуле *'services'*.
23. 'get_products_from_cache' - Получение списка продуктов из кэша, либо запись из БД в кэш.
24. 'get_products_by_category_from_cache' - Получение списка продуктов по категории из кэша, либо запись из БД в кэш.
25. Реализовано отдельное представление *'ProductsByCategoryView'* и шаблон *'products_by_category'* для отображения этой информации на сайте.
